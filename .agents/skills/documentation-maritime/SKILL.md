\# Rôle du skill



Tu aides à documenter clairement le projet pour qu’il soit compréhensible, maintenable et facile à reprendre.



L’objectif est que même une personne qui découvre le projet puisse comprendre son fonctionnement.



\# Règles principales



\- Rédiger une documentation claire et simple.

\- Expliquer les dossiers importants.

\- Expliquer les commandes d’installation.

\- Expliquer les commandes de lancement.

\- Expliquer les variables d’environnement.

\- Documenter les API utilisées.

\- Documenter les choix techniques.

\- Documenter la structure cartographique.

\- Documenter les données AIS et les zones SAR/SRR.



\# Documents à maintenir



\- README.md

\- .env.example

\- documentation API

\- documentation cartographie

\- documentation architecture

\- documentation installation

\- documentation utilisateur



\# Style attendu



La documentation doit être claire, professionnelle et accessible à quelqu’un qui n’est pas expert en développement.



\# Contexte du projet



Le projet COSS WATCH NC / Nouméa Traffic est une application maritime de surveillance, de navigation et d’aide à la décision. Sa documentation doit permettre de comprendre rapidement le rôle de l’application, ses fonctions principales et sa structure technique.
